Allan deviation analysis¶

Allan deviation analysis is a technique for quantifying the different sources of error affecting IMU measurements. The IMU class features methods to fit a model to averaging times \(\tau\) and Allan deviation estimates \(\sigma\).

1# Set up
2import warnings
3import pkg_resources as pkg_rsrc
4import os.path as osp
5import numpy as np
6import xarray as xr
7import matplotlib.pyplot as plt
8import skdiveMove.imutools as imutools
9from scipy.optimize import OptimizeWarning

For demonstrating the methods available in the IMUBase class, IMU measurements from an Android mobile phone were collected for 6 hours at 100 Hz frequency, but subsequently decimated to 10 Hz with a forward/backward filter to avoid phase shift. The phone was kept immobile on a table, facing up, for the data collection period. Note that two sets of measurements for the magnetometer and gyroscope were recorded: output and measured. The type of measurement and the sensor axis constitute a multi-index, which provide significant advantages for indexing, so these are rebuilt:

18icdf = (pkg_rsrc
19        .resource_filename("skdiveMove",
20                           osp.join("tests", "data",
21                                    "samsung_galaxy_s5.nc")))
22s5ds = (xr.load_dataset(icdf)  # rebuild MultiIndex
23        .set_index(gyroscope=["gyroscope_type", "gyroscope_axis"],
24                   magnetometer=["magnetometer_type",
25                                 "magnetometer_axis"]))
26imu = imutools.IMUBase(s5ds.sel(gyroscope="measured",  # use multi-index
27                                magnetometer="measured"),
28                       imu_filename=icdf)

Note that the data collected are uncorrected for bias. It is unclear whether Android’s raw (measured) sensor data have any other corrections or calibrations, but the assumption here is that none were performed.

The first step of the analysis involves the calculation of \(\tau\) and \(\sigma\). The choice of \(\tau\) sequence is crucial.

34maxn = np.floor(np.log2(imu.angular_velocity.shape[0] / 250))
35taus = ((50.0 / imu.angular_velocity.attrs["sampling_rate"]) *
36        np.logspace(0, int(maxn), 100, base=2.0))

These can be used for fitting the ARMAV model and retrieve the coefficients for the five most common sources of variability (quantization, angle/velocity random walk, bias instability, rate random walk, and rate ramp).

37# Silence warning for inability to estimate parameter covariances, which
38# is not a concern as we are not making inferences
39with warnings.catch_warnings():
40    warnings.simplefilter("ignore", OptimizeWarning)
41    allan_coefs, adevs = imu.allan_coefs("angular_velocity", taus)
42
43print(allan_coefs)
