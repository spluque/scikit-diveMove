Can we use an average (median?) rotation matrix? This requires retrieving the direction cosine matrices of the centered data, which can be expressed as Euler angles with respect to the centered-data frame:

109euler_xyz = (imu2whale.orientations["R"]
110             .apply(lambda x: x.as_euler("XYZ", degrees=True))
111             .apply(pd.Series))
112euler_xyz.rename(columns={0: "phi_ctr", 1: "theta_ctr", 2: "psi_ctr"},
113                 inplace=True)
114euler_avg = euler_xyz.mean()
115Rctr2i_avg = R.from_euler("XYZ", euler_avg.values, degrees=True)
116Rctr2i_avg.as_euler("XYZ", degrees=True)
117_euler_ctr2body(Rctr2i_avg)

array([-5.993, +7.107, -7.211])

Check the effect of using this common transformation with the first remaining period:

118idx0 = imu2whale.surface_details.index[0]
119imu_bodyi = imu2whale.orient_surfacing(idx0, Rctr2i_avg)
120# Plot the time series; not bad
121imutools.tsplotIMU_depth(imu_bodyi[_ACCEL_NAME], imu_bodyi["depth"],
122                         figsize=_FIG4X1);

Orient all surface periods with average rotation – note we get a hierarchical dataframe output:

123imu_bodys = imu2whale.orient_surfacings(R_all=Rctr2i_avg)
124# Check out plot of a random sample
125idxs = imu_bodys.coords.get("endasc")  # values in topmost level
126idx_rnd = idxs[np.random.choice(idxs.size)]

128# Compare with period-specific orientation
129Rctr2i = imu2whale.orientations.loc[idx_rnd.to_pandas()]["R"]
130imu_bodyi = imu2whale.orient_surfacing(idx_rnd.to_pandas(), Rctr2i)

Orient the entire IMU object with common rotation:

148gert_frame = imu2whale.orient_IMU(Rctr2i_avg)
